5:T435,<p>Your vault is a graph, but you navigate it through words. When I type <code>[[</code> in Obsidian, what comes to mind determines what gets linked. If I once called something "information overload" and later "consumption trap", those two notes never meet. The API is broken. Consistent vocabulary is consistent wiring. It is the human equivalent of a stable API contract: rename the endpoint and all clients break.</p>
<p>A rich vocabulary does more than help retrieval. It shapes what you <em>can</em> think. If you have precise words for subtle distinctions, you can reason about those distinctions. A programmer with only "list" cannot think in "stack" vs "queue" vs "deque". The same principle applies to personal knowledge. The richer and more precise your naming, the more dimensions you can think in.</p>
