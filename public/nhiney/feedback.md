### 🟠 SERIOUS #4: Bullet points quá dàn trải, thiếu impact cụ thể

Hầu hết bullet points mô tả **những gì đã làm** (task-based), không phải **kết quả / impact** (outcome-based).

**Ví dụ xấu (hiện tại):**
> "Built backend features using Laravel (MVC) with RESTful routing, using Eloquent ORM, Migrations, and Seeders for data management."

**Tốt hơn cần:**
> "Built RESTful API endpoints using Laravel + Eloquent ORM, reducing manual data setup time by ~40% through automated Seeders and Migrations."

**Pattern cần áp dụng:** `[Action] + [Technology] + [Measurable Result]`

Các con số đã có (80+ test cases, 500+ entries, 100+ test cases) là tốt – nhưng cần thêm **context về impact**: tỷ lệ bug giảm bao nhiêu? Coverage đạt bao nhiêu %?

### 🟠 SERIOUS #5: Summary Section quá dài và lặp lại

- Summary nên chỉ cần **3-4 dòng** pitch mạnh, còn lại để Projects tự nói
