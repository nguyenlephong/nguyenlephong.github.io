# Bộ Câu Hỏi Phỏng Vấn Thực Tập Sinh Software Engineer - GreenNode

**Hướng dẫn sử dụng:**  
Bộ câu hỏi này được thiết kế cho buổi luyện phỏng vấn kéo dài 2 giờ, đóng vai nhà tuyển dụng tại GreenNode (một công ty
công nghệ hàng đầu Việt Nam thuộc VNG, tập trung vào Cloud Platform và AI infrastructure). Tôi đã phân loại câu hỏi để
bao quát toàn diện: **kiến thức kỹ thuật**, **kỹ năng thực hành**, **tư duy hệ thống & giải quyết vấn đề**, **văn hóa &
con người**, **học hỏi & phát triển**.

Mỗi câu hỏi kèm **mô tả ngắn gọn** về mục đích: Nhà tuyển dụng muốn verify gì, mong đợi câu trả lời như thế nào (dựa
trên JD của GreenNode, nhấn mạnh mentorship, backend/cloud, learning mindset).

**Cấu trúc buổi luyện (2h):**

- 0-30p: Giới thiệu & Văn hóa/Con người (ice-breaker).
- 30-90p: Kỹ năng & Kiến thức kỹ thuật (core).
- 90-110p: Tư duy hệ thống & Problem-solving.
- 110-120p: Học hỏi & Q&A (kết thúc).

In ra file Word để em bạn luyện: Đọc câu hỏi to, ghi chép câu trả lời, sau đó feedback dựa trên "Mong đợi".

---

## 1. Văn Hóa & Con Người (15-20 câu hỏi, 30 phút)

Những câu hỏi này verify **fit văn hóa GreenNode** (teamwork, openness to feedback, collaboration trong môi trường VNG
Campus năng động, mentorship-heavy).

- **Câu hỏi:** Hãy kể về bản thân bạn và lý do ứng tuyển vị trí Software Intern tại GreenNode?  
  **Mục đích:** Verify động lực cá nhân, sự phù hợp với sứ mệnh Cloud/AI của GreenNode. Mong đợi: Nêu rõ đam mê
  backend/cloud, biết về VNG/GreenNode (research công ty), thể hiện curiosity về distributed systems.

- **Câu hỏi:** Bạn làm việc nhóm như thế nào trong dự án lớn nhất của bạn?  
  **Mục đích:** Check collaboration skills (nice-to-have). Mong đợi: Ví dụ cụ thể về teamwork, openness to feedback, Git
  workflow chia sẻ code.

- **Câu hỏi:** Làm thế nào bạn xử lý khi nhận feedback tiêu cực từ mentor/senior?  
  **Mục đích:** Verify willingness to learn under mentorship. Mong đợi: Thái độ tích cực, ví dụ cải thiện sau feedback,
  không phòng thủ.

- **Câu hỏi:** Bạn nghĩ văn hóa "ask questions" quan trọng thế nào trong engineering?  
  **Mục đích:** Check strong learning mindset (must-have). Mong đợi: Đồng tình, ví dụ hỏi để debug/learn nhanh.

- **Câu hỏi:** Tại sao bạn muốn làm việc tại VNG Campus, HCMC 3 ngày/tuần?  
  **Mục đích:** Verify commitment & fit môi trường. Mong đợi: Nhấn mạnh lợi ích mentorship, gym/pool như perks, sẵn sàng
  full-time sau.

- **Câu hỏi:** Bạn đã từng đóng góp ý tưởng cải thiện dự án chưa? Kết quả ra sao?  
  **Mục đích:** Check proactivity & teamwork. Mong đợi: Ví dụ propose tech mới, ảnh hưởng tích cực đến team.

---

## 2. Kiến Thức Cơ Bản & OOP (15 câu hỏi, 30 phút)

Verify **solid understanding of programming fundamentals & OOP** (must-have), basic backend concepts.

- **Câu hỏi:** Giải thích 4 nguyên tắc OOP cơ bản (Encapsulation, Inheritance, Polymorphism, Abstraction).  
  **Mục đích:** Verify nền tảng OOP. Mong đợi: Định nghĩa rõ, ví dụ code đơn giản (Java/Python).

- **Câu hỏi:** Sự khác biệt giữa interface và abstract class? Khi nào dùng cái nào?  
  **Mục đích:** Check depth OOP. Mong đợi: Biết multiple inheritance issue, ví dụ backend service.

- **Câu hỏi:** Viết pseudocode cho một REST API endpoint đơn giản (GET user by ID).  
  **Mục đích:** Verify basic backend concepts. Mong đợi: Hiểu HTTP methods, JSON response, error handling.

- **Câu hỏi:** Git workflow cơ bản của bạn là gì? (branch, commit, PR, merge).  
  **Mục đích:** Must-have Git familiarity. Mong đợi: Mô tả Git Flow hoặc GitHub Flow, ví dụ thực tế.

- **Câu hỏi:** Giải thích HTTP status codes 200, 404, 500.  
  **Mục đích:** Basic backend knowledge. Mong đợi: Liên hệ với reliability/scalability trong JD.

- **Câu hỏi:** Bạn quen ngôn ngữ lập trình nào nhất? Tại sao?  
  **Mục đích:** Check programming fundamentals. Mong đợi: Java/Python/Go (phù hợp backend GreenNode), ví dụ dự án.

---

## 3. Kỹ Năng Thực Hành & Backend Development (12 câu hỏi, 30 phút)

Verify **ability to write code, debug, code review** dưới mentorship, focus performance/reliability.

- **Câu hỏi:** Làm thế nào để optimize một backend API chậm? Các bước debug?  
  **Mục đích:** Check problem-solving systematically (nice-to-have). Mong đợi: Bước: profile, indexing DB, caching; ví
  dụ tools (Postman, logs).

- **Câu hỏi:** Bạn đã deploy code lên production chưa? Quy trình như thế nào?  
  **Mục đích:** Verify software release workflow. Mong đợi: CI/CD basics (GitHub Actions), testing trước deploy.

- **Câu hỏi:** Giải thích code review: Bạn check những gì ở code của người khác?  
  **Mục đích:** Check participation in team rituals. Mong đợi: Correctness, readability, best practices (SOLID).

- **Câu hỏi:** Xử lý exception trong backend service như thế nào để reliable?  
  **Mục đích:** Focus reliability/scalability. Mong đợi: Global exception handler, logging, graceful degradation.

- **Câu hỏi:** Pseudocode: Implement một simple cache (LRU) với HashMap.  
  **Mục đích:** Hands-on coding under mentorship. Mong đợi: Logic evict least recently used.

- **Câu hỏi:** Bạn đọc technical English docs như thế nào? Ví dụ gần nhất?  
  **Mục đích:** Must-have technical English. Mong đợi: Đọc AWS/Docker docs, áp dụng vào dự án.

---

## 4. Tư Duy Hệ Thống & Problem-Solving (10 câu hỏi, 20 phút)

Verify **systems thinking, engineering mindset** cho cloud-native/distributed systems.

- **Câu hỏi:** Giải thích scalability trong backend: Horizontal vs Vertical?  
  **Mục đích:** Check basic cloud concepts (GreenNode focus). Mong đợi: Load balancer, sharding ví dụ.

- **Câu hỏi:** Một service down làm toàn hệ thống crash? Làm sao tránh?  
  **Mục đích:** Verify distributed systems awareness. Mong đợi: Circuit breaker, microservices isolation.

- **Câu hỏi:** Debug một bug production: Các bước systematic?  
  **Mục đích:** Willingness to debug (must-have). Mong đợi: Logs > reproduce > root cause > fix > test.

- **Câu hỏi:** Nice-to-have: Kubernetes cơ bản là gì? Bạn đã thử chưa?  
  **Mục đích:** Learning agility cho cloud-native. Mong đợi: Biết container orchestration, eagerness to learn.

- **Câu hỏi:** Problem: Tối ưu query DB chậm với 1M records.  
  **Mục đích:** Engineering mindset (performance). Mong đợi: Index, pagination, read replicas.

---

## 5. Học Hỏi & Phát Triển (8 câu hỏi, 10 phút + Q&A)

Verify **curiosity, learning agility**, path to full-time.

- **Câu hỏi:** Công nghệ mới bạn đang học? Tại sao liên quan GreenNode?  
  **Mục đích:** Curiosity & propose improvements. Mong đợi: Docker, Kafka, AI infra; liên hệ JD.

- **Câu hỏi:** Bạn mong đợi gì từ mentorship tại GreenNode?  
  **Mục đích:** Check growth mindset. Mong đợi: Hands-on backend, exposure production systems.

- **Câu hỏi:** Điểm yếu kỹ năng của bạn? Làm sao cải thiện?  
  **Mục đích:** Openness to feedback. Mong đợi: Cụ thể (e.g. distributed systems), plan học (courses/projects).

- **Câu hỏi:** Bạn có câu hỏi nào cho chúng tôi về GreenNode không?  
  **Mục đích:** Verify interest. Mong đợi: Hỏi về team structure, projects thực tế, full-time path.
